BA Change Workflow
End-to-end change request lifecycle — Request → Approval → Development → Testing → Closure
Stage 1
📝
Request
Business need identified and change request formally raised by stakeholder or BA
Stage 2
✅
Approval
Change Review Board (CRB) evaluates impact, risk and feasibility before authorising
Stage 3
⚙️
Development
Approved change built and implemented by development team per agreed specifications
Stage 4
🧪
Testing
QA team validates the change in UAT environment; BA verifies against acceptance criteria
Stage 5
🏁
Closure
Change deployed to production, stakeholders sign off, documentation updated and CR closed
Stage Details & Responsibilities
📝 Request
Stakeholder raises business need
BA captures requirements
CR form completed with impact, scope, priority
Initial effort estimate attached
CR assigned unique ID and logged
✅ Approval
CRB reviews risk and feasibility
IT Manager signs off for Technical CRs
Budget approval for high-cost items
Compliance review for Regulatory CRs
Status updated to Approved or Rejected
⚙️ Development
Dev team receives approved spec
Sprint planning and task breakdown
Code review and peer validation
BA reviews mid-sprint if scope changes
CR status: In Progress
🧪 Testing
QA executes test cases in UAT
BA runs acceptance testing
Defects raised, tracked and resolved
Regression testing for critical CRs
Sign-off by stakeholder required
🏁 Closure
Production deployment scheduled
Rollback plan confirmed
Go-live notification sent to stakeholders
Post-implementation review conducted
CR status set to Done and archived

Escalation & Exception Rules
Escalation Triggers
⚡Critical CRs not approved within 24 hours → auto-escalate to IT Director
⚡Any CR in Development >21 days → flag to Project Manager
⚡UAT defects unresolved after 3 days → escalate to Dev Lead
⚡No stakeholder sign-off after 5 days → re-schedule closure review
Exception Handling
🔄Emergency change (P0): bypass CRB, get verbal approval, document post-facto
🔄Scope creep detected: raise new CR, do not expand existing one
🔄Rejected CRs may be resubmitted after addressing CRB feedback
🔄Regulatory CRs require compliance officer signature before closure
