#  BA Change Tracker System  
**By Aishwarya Rajeshirke**

---

##  Project Overview
The BA Change Tracker System is a web-based application designed to manage, track, and analyze change requests across an organization.

This project demonstrates how a Business Analyst translates business requirements into a functional system for tracking changes, improving transparency, and enabling data-driven decision-making.

---

##  Problem Statement
In many organizations, change requests are:
- Scattered across emails and Excel files  
- Difficult to track across stakeholders  
- Lacking visibility on status, priority, and ownership  

This leads to:
- Delays in delivery  
- Poor stakeholder communication  
- Missed critical changes  

---

## Solution
This system provides a centralized platform to:
- Log and manage change requests  
- Track status and priority  
- Monitor workflow stages  
- Generate insights via analytics dashboard

##  Key Features

### Change Request Management
- Create and manage change requests  
- Unique Change ID tracking  
- Fields include:
  - Title  
  - Description  
  - Priority (Low, Medium, High, Critical)  
  - Status (Open, In Progress, Review, Done, Rejected)  
  - Owner  
  - Impact Area  

---

### Dashboard & KPIs
- Total Changes  
- Active Changes  
- Completed Changes  
- Critical Changes  

---

### Analytics
- Status Distribution  
- Changes by Priority  
- Changes by Owner  
- Monthly Trend Analysis  

---

### Workflow Lifecycle
1. Request  
2. Approval  
3. Development  
4. Testing (UAT)  
5. Closure  

---

### SLA & Escalation Rules
- Approval SLA: 5 Days  
- Development SLA: 14 Days  
- UAT SLA: 5 Days  

Escalations:
- Critical CR not approved within 24 hrs  
- Development exceeding timeline flagged  

---

### Reporting
- Export data to CSV  
- Generate reports:
  - Executive Summary  
  - Change Log  
  - Risk Assessment
